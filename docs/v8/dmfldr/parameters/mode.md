---
title: MODE
---

dmfldr 的装载模式，字符串类型。缺省值为 `IN`。`MODE` 的可选项有 `IN`、`OUT`、`OUTORA` 和 `LOCAL_FILE` 四种：

- `IN` 模式：从数据文件中将数据装载入数据库，这种模式下控制文件的格式对应为数据文件中现有数据的格式；
- `OUT` 模式：从数据库中将数据导出到数据文件，这种模式下控制文件所指定的格式为数据存放在数据文件中的格式。需要说明的是在 `OUT` 模式下，如果指定了多个数据文件，则 dmfldr 最终只会将数据写入第一个数据文件。`OUT` 模式下的控制文件与 `IN` 模式下的控制文件格式相同，用户可以通过使用同一个文件进行载入和导出数据。
- `OUTORA` 模式：表示导出 ORACLE 表的数据，此模式下暂不支持带有大字段表的导出。
- `LOCAL_FILE` 模式：表示数据导入时，不向服务器发送数据，而是将数据写入成本地文件。通常用于多站点场景下，将数据按站点分类重新组织数据。
