---
title: "初始化实例时报错：\"dminit: symbol lookup error: dminit: undefined symbol\""
---

**【问题描述】**

初始化实例是报错：`dminit: symbol lookup error: dminit: undefined symbol: rlog4_pkg_crc_stand_set`

**【问题分析】**

初始化实例报错一般两种原因

- 安装的数据库与操作系统 CPU 不符。

- 本地装有多个达梦数据库，该目录下 `dminit` 使用的 so 文件不是自身版本。

**【问题解决】**

- 如果数据库版本与当前操作系统 CPU 不符合，则需要更换数据库版本。

- 执行 ldd `dminit` 命令检查依赖的库文件是自身的还是旧库的，命令如下：
```
ldd /数据库安装目录/bin /dminit
```

如果是由于多个达梦数据库版本存在，新安装的数据库依赖的的库文件为旧库的库文件导致无法初始化实例，可以通过修改环境变量，注释掉之前的版本来解决该问题。

```
##修改环境变量
vim ~/.bash_profile
export LD_LIBRARY_PATH="$LD_LIBRARY_PATH:/data/dmdbms/bin"
export DM_HOME="/data/dmdbms"
##使环境变量生效
source ~/.bash_profile
```
