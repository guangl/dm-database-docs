---
title: "Windows 数据库卸载不完整，服务/实例名被占用，怎么解决？"
---

**【问题描述】**

Windows 数据库卸载不完整，服务/实例名被占用，怎么解决。

**【问题原因】**

数据库卸载程序默认不会自动清理已注册的 Windows 系统服务，如果卸载前服务未手动删除，对应的服务名会在系统中残留，导致重新安装时服务/实例名被占用。

**【问题解决】**

- 数据库卸载默认不会删除 data 目录，这是为了保护数据，如确定不保留历史数据，可由管理员手动删除。

- 数据库自带的达梦数据库服务查看器里里查看是否有 `DmService+` 服务名 的服务注册。

- 管理员开启 `CMD` 命令行界面，然后执行：`sc delete "服务名"`。例如：`sc delete DmServiceDMSERVER`。
