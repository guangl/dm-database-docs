---
title: "[-7103]: 创建命名管道失败"
---

**【问题描述】**

中标麒麟 7 环境，在将达梦数据库 bin 路径配置到环境变量 path 中，可以在任意目录下执行 dmrman 命令，但是进入 dmrman 后执行备份命令，会有如下报错：`[-7103]: 创建命名管道失败`。

**【问题原因】**

当执行 dmrman 命令进行备份操作的时，会根据 dmrman 命令所在路径下寻找由 `DM_PIPE_` 开头的管道文件，而不是通过 path 设置的环境变量路径来寻找对应的管道文件，`DM_PIPE_` 一般是在 bin 目录下生成。

**【问题解决】**

**方法一：**

使用达梦数据库安装用户 `dmdba` 进入达梦数据库安装路径 `.../bin` 目录下启动 dmrman 工具，执行备份脚本提示成功。

**方法二：**

达梦数据库备份还原依赖 dmap 服务，必须启动，且一定要正确的用户启动。

1. 如果 dmserver 是 `dmdba` 用户启动的，则 dmap 也要用 `dmdba` 用户启动。
2. 前端命令启动方式：

```bash
cd /home/dmdba/dmdbms/bin
./dmap
```

3. 也可通过操作系统服务的方式启动：

```bash
systemctrl start DmAPService
```

否则会报错，如下所示：

```plaintext
os_pipe2_create_server_with_name, name:[ /DM_PIPE_DMRMAN-3676_1_RD], error:[2]
```

假如 1、2 或者 1、3 都无问题，依旧报错：

```plaintext
os_pipe2_create_server_with_name, name:[ /DM_PIPE_DMRMAN-3676_1_RD], error:[2]
```

可进行以下操作：

```bash
cd /home/dmdba/dmdbms/bin
rm -rf DM_PIPE_*
```

完成后再依次进行 1、2 或者 1、3 操作步骤。
