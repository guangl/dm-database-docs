---
title: "怎么配置归档，有哪些注意事项？错误号 [-8216]"
---

**【问题描述】**

报错：`[-8216]: 指定或者默认归档目录中找不到归档日志，缺少本地或者远程归档`。

**【问题原因】**

数据库未正确配置归档，导致指定或默认归档目录中缺少本地或远程归档日志。

**【问题解决】**

达梦数据库开启归档方式：

SQL 方式开启归档：

```sql
alter database mount;
alter database add  archivelog 'dest=/d07,TYPE=local,FILE_SIZE=1024,SPACE_LIMIT=200000';
alter database archivelog;
alter database open;
```

图形界面方式开启归档：

选择【连接】，右键【管理服务器】，进入系统管理页面，选择【配置】，点击【转换】。进入归档配置页面，归档模式选择【归档】，添加归档参数，填写归档目录，选择归档类型，填写文件大小，空间大小限制（注意归档路径可以有多个）。返回系统管理页面，状态转换选择【打开】，重新打开数据库。

编辑配置文件方式开启归档：

首先修改 `dm.ini` 文件的 `arch_ini` 参数为 1，然后在 `dm.ini` 文件目录新建 `dmarch.ini` 文件，添加以下内容：

```plaintext
ARCH_WAIT_APPLY   = 1
[ARCHIVE_LOCAL1]
ARCH_TYPE         = LOCAL
ARCH_DEST         = /d07
ARCH_FILE_SIZE       = 64
ARCH_SPACE_LIMIT     = 0
```

重启数据库即可开启归档模式。需要注意配置 `ARCH_SPACE_LIMIT` 参数，为 0 是不限制归档大小，其它数值为限制归档量大小，单位是 MB。
