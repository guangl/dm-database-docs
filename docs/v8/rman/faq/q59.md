---
title: "备份报错：无效的 TRACE 文件"
---

**【问题描述】**

执行备份命令时报错：

```plaintext
CODE:[-8086], DESC:[无效的TRACE文件[../log/dm_SBTTRACE_202306.log]], COST:[00:00:00]
```

**【问题原因】**

此问题原因为 dmap 进程运行过程中 bin 目录或者产品被重装过导致，具体流程为：

1. dmap 进程正常运行；
2. 删除数据库软件目录；
3. 对数据库软件进行重新安装；
4. 此时再对数据库进行 backup 备库就会报错。

**【问题解决】**

可以通过以下两种方法解决：

**方法一：**重新启动 dmap 服务进行备份。

**方法二：**使用非 dmap 进程进行备份，具体方法如下：

1. 执行以下命令修改参数 `BAK_USE_AP` 值为 2，不依赖 dmap，由主进程 dmserver 自身执行备份还原，但不支持第三方备份：

```sql
SP_SET_PARA_VALUE(1,'BAK_USE_AP',2);
```

2. 重新执行备份。
